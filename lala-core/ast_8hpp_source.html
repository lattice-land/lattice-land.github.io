18 /** We call an "abstract variable" the representation of this variable in an abstract domain.
19 It is simply an integer containing the UID of the abstract element and an internal integer variable identifier proper to the abstract domain.
20 The mapping between logical variables and abstract variables is maintained in `Environment` below.
21 An abstract variable always has a single name (or no name if it is not explicitly represented in the initial formula).
22 However, a logical variable can be represented by several abstract variables when the variable occurs in different domains. */

29 /** The approximation of a formula in an abstract domain w.r.t. the concrete domain.
30 * `UNDER`: An under-approximating element contains only solutions but not necessarily all.
31 * `OVER`: An over-approximating element contains all solutions but not necessarily only solutions.
32 * `EXACT`: An exact element is both under- and over-approximating; it exactly represents the set of solutions. */
